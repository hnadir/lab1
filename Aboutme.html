Academic Projects

  • Client/Server Configuration Chat Room Facility in Python Jun'15

    (Semester Project for Computer Networks)
    • Designed and implemented own protocol to run chat message exchange in round robin fashion, where each participant is given a turn to type their message.
    • If person on-turn has nothing to say, a timeout mechanism passes the token onto next participant.
    • The overall chat management is done at server side, where server sends messages to all participating clients.
    • The chat room supports up to five clients, informing surplus clients to attempt a connection later. If only one person is in the chat room, they are informed to wait till someone else joins the session. Once someone joins the chat room, a message informing about the new user is sent to all participants.
